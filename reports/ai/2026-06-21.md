# 🤖 LLM 推理引擎趋势分析

> 📅 2026-06-19 ~ 2026-06-21

## 一、核心趋势（3-5 条）

- **SGLang 开发活跃度领先**：在三天内，SGLang 以 70 个 Commits 和 109 个合并 PR 的绝对数量领先于其他仓库，显示出当前最高的开发与迭代速度。其合并/提交比 (109/70) 极高，表明其协作流程高效。
- **DeepSeek V4 成为行业焦点**：所有四个主流推理引擎（vLLM、SGLang、TensorRT-LLM、llama.cpp）都在本周内有直接或间接相关的活动。llama.cpp 和 SGLang 社区围绕 DeepSeek V4 的 Roadmap 和模型支持讨论热烈，这已成为当前最受关注的模型适配任务。
- **KV Cache 量化与通信优化是两大技术热点**：vLLM 和 llama.cpp 的 Issues 中均出现了对 **TurboQuant** KV Cache 量化的新功能请求，显示出对内存压缩技术的共同追求。同时，SGLang 和 TensorRT-LLM 都在 PR 中重点优化 MoE 模型的 Expert-Parallel **All-to-All 通信**（使用 NVSHMEM 或 WideEP），以降低分布式推理延迟。
- **SGLang 主导架构演进**：与其他仓库相比，SGLang 的提交类型中包含了显著的 `refactor:9` 和 `api:1`，表明其不仅在进行功能性开发，也在主动进行内部架构重构和API层面演进，以支撑未来更复杂的功能（如 Diffusion LLM）。

## 二、各仓库要点

### vllm-project/vllm
- **关键变更**：共 35 个 Commits，主要专注于 Bug 修复 (18个)，特性开发 (6个) 和性能优化 (4个)。合并了 68 个 PR。
- **值得关注的**：
    - **社区讨论热烈**：`TRITON_MLA_SPARSE` Kernel 优化和新硬件支持 (PR #38476) 获得 156 条评论，显示出社区对扩展硬件兼容性的高度兴趣。
    - **架构重构**：通过 `Composite model loading` (Issue #15697) 和 `Canonical KV Cache Allocation` (PR #37885) 等议题，vLLM 正在对模型加载和 KV 缓存分配等核心架构进行重大改进。
    - **路线图明确**：发布了 Q2 2026 的 **vLLM × RL 路线图**，明确将强化学习工作负载（训练与推理）作为下一阶段的一等公民。

### sgl-project/sglang
- **关键变更**：以 70 个 Commits 领跑，其中 Bug 修复最多 (22个)，同时包含大量特性 (12个) 和性能优化 (8个)。合并了 109 个 PR。
- **值得关注的**：
    - **前瞻性技术布局**：发布了 **DeepSeek V4** (Issue #23602) 和 **Diffusion LLMs** (Issue #14199) 的路线图，为未来半年到一年的新产品和技术方向做好准备。
    - **底层性能创新**：提出使用 **NVSHMEM** 替代 NCCL/DeepEP 进行 All-to-All 通信 (Issue #22583)，旨在实现极低延迟的 Expert 分发。
    - **新硬件与新模型**：正在积极添加对 **Ascend NPU** (PR #25144) 和 **MiniMax-M3** (PR #27944) 的支持，显示其拓展硬件和模型生态的魄力。

### NVIDIA/TensorRT-LLM
- **关键变更**：Commits 仅 3 个，活动量最低。合并了 11 个 PR。
- **值得关注的**：
    - **深耕分布式与模型分片**：PR #14786 和 #15222 专注于 Sharding IR 和 Disaggregated Transceiver 的深度集成，旨在提升大型 MoE 模型的自动分片能力和指令-解码分离架构性能。
    - **底层算子优化**：PR #13404 针对 NVLink AlltoAll 内核增加 `active_rank_mask`，精确优化 MoE 的通信模式。
    - **新量化格式跟进**：Issue #10241 请求为 Blackwell 架构 (SM120) 添加 NVFP4 KV Cache 支持，持续跟进最新量化技术。

### ggerganov/llama.cpp
- **关键变更**：共 24 个 Commits，主要包括特性开发 (12个) 和 Bug 修复 (7个)。合并了 54 个 PR，合并效率高。

    - **持续跟进前沿模型**：正在进行的 **DeepSeek V4** (PR #24162) 和 **Step3.5/3.7 flash mtp3** (PR #24340) 模型支持 PR，紧跟行业模型更新节奏。
    - **API 能力增强**：PR #21174 致力于提升 Responses API 的合规性与 Codex CLI 的兼容性，表明它在持续向 OpenAI API 标准看齐，以服务更广泛的工具链。
