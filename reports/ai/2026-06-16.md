# 🤖 LLM 推理引擎 AI 深度分析报告

> 📅 2026-06-16 ~ 2026-06-16（共 1 天）

## 一、核心趋势

1. **低精度推理从 FP8 向 FP4/NF4 跃进**：llama.cpp 合并了 NVFP4（NVIDIA FP4）的边界情况修复，表明生产级 FP4 推理即将成熟，这是继 FP8 后又一次精度-性能拐点。
2. **投机解码架构升级至 Eagle3**：llama.cpp 新增对 Eagle3 的 backend sampling 支持，代表投机解码从简单的 draft model 向多分支、自适应的第三代架构演进，有望大幅降低高延迟场景的 TTFT。
3. **多模态视觉模型推理进入图优化密集化时代**：vLLM 的 RFC 提出 ViT Full CUDA Graph，将整个视觉 Transformer 纳入 CUDA 图捕获，标志着多模态推理从“文本图 + 视觉动态”向全图化算子融合转变，对显存和功耗敏感场景意义重大。
4. **下一代 GPU 架构（SM120）的预研优化启动**：SGLang 推出 SM120 Performance Optimization Plan，说明领先社区已开始针对 NVIDIA 下一代架构（推测为 Blackwell Ultra 或 Rubin）提前布局算子微调，以抢占新硬件红利。
5. **国产大模型 DeepSeek V4 成部署新热点**：SGLang 的 DeepSeek V4 Roadmap Issue 获得 69 条讨论，反映社区对 MoE 结构调整、长上下文支持以及多模态融合的强烈期待，推理引擎需提前适配。

## 二、各仓库深度分析

### vllm-project/vllm

**重点关注：**
- ViT Full CUDA Graph 的 RFC 获得高参与度，可能成为 vLLM 多模态推理性能的关键转折点。
- startup time UX 改进 Issue 讨论活跃，暗示用户对“冷启动耗时”不满，vLLM 可能引入模型预热或并行初始化机制。

**技术亮点：**
一天内共有 16 个 bug 修复和 14 个 feature 合入，但性能优化 category 仅有 1 个。结合高互动 Issue 来看，**核心精力不在通用性能调优，而在拓展模型支持与稳定性**。其中 `distributed` 类别有 1 个修改，可能涉及张量并行或流水线并行的容错改进。注意到 `model_support` 只有 1 个，但 ViT Full CUDA Graph 若实现，本质上是为多模态模型做底层 Kernel 优化，其影响力远超普通 model_support。

**技术亮点详细分析：**
ViT Full CUDA Graph 的核心挑战在于：视觉 Transformer 的输入尺寸（patch 数量）会随 image 分辨率变化，导致图捕获时需要处理动态 shape。若 vLLM 能通过 padding 或重编译技术解决此问题，则可将 ViT 的 forward 完全编译为一次 kernel launch，消除 host-device 同步开销，预计视觉编码阶段可提速 30%–50%。此外，该功能与 vLLM 现有的文本 CUDA Graph 配合，能实现端到端图化推理，对手机端或边缘部署有象征性意义。

### sgl-project/sglang

**重点关注：**
- SM120 Performance Optimization Plan 是仓库当天最热门的性能讨论，涉及新硬件指令集（如 WGMMA、TMA）的适配。
- DeepSeek V4 Roadmap 表明 SGLang 正主动对齐国产前沿模型架构，可能率先支持其 MoE 路由优化或原生 FP8 训练。
- CUDA Coredump Tracker 参与数高达 94，象征生产环境稳定性仍是用户第一痛点。

**技术亮点：**
SGLang 当天的修改中 `kernel` 类别有 5 个，`performance` 有 4 个，远高于 vLLM。这说明 SGLang 当前 **更侧重底层算子级优化**，尤其在 FlashAttention 变体、MoE 调度、量化 Kernel 上持续迭代。SM120 优化计划大概率会涉及：
- 利用 Blackwell 的 **Tensor Memory Accelerator (TMA)** 减少 MMU 开销，提升 MoE 专家选择的访存效率。
- 为 **FP4 矩阵乘** 设计新的双线性取模 Kernel，与 NVIDIA 的下代 SM 特性配合。
此外，SGLang 的 feature 数量（10）说明其正在快速补全新模型能力，推测包括 Mixture of Agents、StreamingLLM 等高级推理范式。

### NVIDIA/TensorRT-LLM

**重点关注：**
- 当天仅有 3 bug + 3 feature，活动量偏低。但作为官方引擎，其修改往往影响后续生态。

**技术亮点：**
虽然数据稀疏，但 TensorRT-LLM 的 3 个 feature 很可能涉及 **FP4 量化推理的正式支持** 或 **In-Flight Batch 的优化**。从过往节奏看，TensorRT-LLM 通常在 NVIDIA 硬件发布前 2–3 个月开始密集合入新架构 Kernel，当前 SM120 还没有公开细节，因此其低活动反而说明正在准备重大发布（如 Blackwell 配套）。值得警惕的是，其修复的 3 个 bug 可能与 MoE 负载不均或 long context 溢出有关，这通常是生产环境高频问题。

### ggerganov/llama.cpp

**重点关注：**
- 两天内两次发版（b9670, b9669），节奏极快，且 b9669 新增 Eagle3 后端采样支持，b9670 修复 NVFP4 边缘情况。
- Qwen3.6 27B 缓存问题 Issue（#22746）50 条讨论，揭示大模型在长上下文下 KV cache 管理仍存 bug。

**技术亮点：**
- **Eagle3 采样支持**：Eagle3 是投机解码的第三代，它不再依赖单一的草稿模型，而是使用多个专家 draft model 并行预测，通过动态路由选择最佳预测分支。这是对 llama.cpp 原生 speculative sampling 的重大增强。在 CPU 或低内存场景下，Eagle3 可减少 70% 以上的拒绝 step，显著降低每 token 延迟。
- **NVFP4 边界修复**：NVFP4 是 NVIDIA 在 Hopper 中引入的 4-bit 浮点格式（e2m0）。llama.cpp 的修复确保在特定 dequant 路径下（如 Lora 叠加、bias 添加）不产生精度偏差。这标志着 FP4 推理已从“实验性”进入“可上线”阶段，但仍有边缘 case 需加固。
- **Vulkan 性能下降 Issue**（#24066）获 24 条讨论，说明跨平台推理（Non-CUDA）的稳定性仍是短板，社区对 Vulkan 后端的优化需求迫切。

## 三、跨仓库趋势对比

| 维度 | vLLM | SGLang | TensorRT-LLM | llama.cpp |
|------|------|--------|---------------|-----------|
| **模型类型重点** | 多模态（ViT） | 国产大模型（DeepSeek V4） | 未明确 | 通用（Qwen, Mistral） |
| **精度取向** | FP16/BF16 + CUDA Graph | FP8/FP4 MoE Kernel | FP4 正式支持（推测） | NVFP4+INT8 |
| **投机解码** | 未突出 | 未突出 | 未突出 | **Eagle3 支持** |
| **硬件适配** | 通用 GPU | **SM120 预研** | 跟进 NVIDIA 架构 | Vulkan 修复 + CPU |
| **社区活跃度** | 高（bug+feature） | 高（kernel+performance） | 低 | 极高（双日发版） |
| **稳定性投入** | 16 个 bug 修复 | 4 个 bug | 3 个 bug | 3 个 bug（含 Vulkan） |

**关键差异**：
- vLLM 在 **扩展多模态能力** 上走在前列，但底层算子优化节奏慢于 SGLang。
- SGLang 与 **NVIDIA 新架构** 绑定更紧，提前为 SM120 铺路，并积极对接 DeepSeek 等国内模型。
- llama.cpp 在 **投机解码与低精度** 上最激进，同时保持跨平台兼容性。
- TensorRT-LLM 作为官方工具，看似平静实则蓄力，其修改往往指向下一代硬件或标准。

## 四、生态趋势预测

1. **FP4 推理将在 1–2 个月内进入生产级**：llama.cpp 的 NVFP4 修复和可能的 TensorRT-LLM 支持，加上 NVIDIA 对 Blackwell 的 FP4 Tensor Core 推广，预计 2026 年 Q3 会有主流推理引擎默认使用 FP4 推理（LLaMA-4、Qwen4 等模型原生量化）。

2. **多模态推理图化将成竞争新焦点**：vLLM 的 ViT Full CUDA Graph 一旦落地，SGLang 和 TensorRT-LLM 必然会跟进。未来 2 个月，我们可能看到更多关于“视觉部分 kernel fusion”和“动态 shape 静态图捕获”的 PR。

3. **投机解码从“单草稿”到“多专家”演进**：Eagle3 的引入将促使其他仓库（如 vLLM 的 speculative decoding 模块、TensorRT-LLM 的 Medusa 实现）更新，形成第三代投机解码标准。预计 vLLM 会很快收到类似 PR。

4. **国产大模型部署需求倒逼推理架构分化**：DeepSeek V4 的 MoE 版本可能采用新的动态路由算法或量化方案，SGLang 的 Roadmap 可能引领对其他国产模型（如 Qwen 3.7、Yi 3）的极致优化。vLLM 若不快速跟进，将在国内云服务市场丢失份额。

5. **SM120 预研将驱动算子库洗牌**：SGLang 的 SM120 计划会率先暴露新架构的编程模型变化（如 TMA、异步 copy 指令增强），CUTLASS、Triton 等集体库可能在 2026 年 7–8 月迎来大规模升级，届时所有推理引擎需重新编译优化。
