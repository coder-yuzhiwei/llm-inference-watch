# 🤖 LLM 推理引擎趋势分析

> 📅 2026-06-21 ~ 2026-06-16（共六天）

## 一、核心趋势

1. **Bug 修复与功能开发并重**：四个仓库 commit 总数中 bug 修复占 135，feature 占 104，性能/模型支持/内核优化合计 74，表明社区在稳定性的同时积极拓展新能力。
2. **前沿模型迅速跟进**：vllm、sglang、llama.cpp 均出现对 DeepSeek V4 系列的支持或路线图讨论，多模态 MoE 架构成为焦点。
3. **稀疏注意力与量化持续发力**：vllm 提出 TRITON_MLA_SPARSE backend 支持多代 GPU；llama.cpp 请求 TurboQuant 支持；TensorRT-LLM 推进 NVFP4 KV cache 覆盖 Blackwell 平台。
4. **硬件适配向新架构延伸**：AMD 在线量化（vllm）、Blackwell SM120 数值格式（TensorRT-LLM）说明推理引擎正为新一代硬件预研优化。
5. **Agentic 推理路径初现**：sglang 提出 Programmatic KV Cache 用于代理式工作负载，呼应 AI Agent 场景对内存管理和多轮交互的差异化需求。

## 二、各仓库要点

### vllm-project/vllm
- **关键变更**：149 个 commits 中 bug 修复 65，feature 34；合入 PR 255 个。值得注意的 feature 包括 AutoWeightsLoader 统一模型加载、AMD-Quark 在线量化 RFC。
- **值得关注的**：PR #38476 新增 TRITON_MLA_SPARSE backend，支持 SM80/11x/12x 上的稀疏 MLA 注意力，可运行 DeepSeek-V3.2 等模型。

### sgl-project/sglang
- **关键变更**：157 个 commits 中 bug 修复 48，feature 36，kernel 优化 15；合入 PR 269 个。社区对 DeepSeek V4 有完整 roadmap。
- **值得关注的**：Issue #27574 提出 Programmatic KV Cache 方案，从引擎上层管理 KV 块以优化 Agent 推理场景，具有前瞻性。

### NVIDIA/TensorRT-LLM
- **关键变更**：25 个 commits 中 feature 13，bug 7；合入 PR 77 个。本期没有大规模重构，以增量功能为主。
- **值得关注的**：Issue #10241 请求在 trtllm-gen 中为 SM120（Blackwell）增加 NVFP4 KV cache 支持，说明团队正为消费级新硬件提前布局。

### ggerganov/llama.cpp
- **关键变更**：48 个 commits 中 feature 21，bug 15，performance 4；合入 PR 114 个。社区活跃，issue 数量达 347，主要来自模型请求和功能建议。
- **值得关注的**：Issue #22319 要求支持 DeepSeek V4 系列；#20977 请求 TurboQuant（量化 & KV cache 优化），反映社区对 4-bit 量化和高效内存管理的强烈需求。

## 三、下月展望

- **模型支持竞赛**：随着 DeepSeek V4 等超大 MoE 模型开源，四引擎会加速适配，尤其是 Sparse MLA 和 FP4 量化路径。
- **稀疏与量化深度融合**：TRITON_MLA_SPARSE、TurboQuant、NVFP4 等技术可能从实验性 PR 走向稳定合并，推动单卡及多卡吞吐量再上台阶。
- **Agent 场景专用优化**：sglang 的 Programmatic KV Cache 可能引起更多讨论与实践，其他引擎或借鉴类似思路。
- **硬件兼容性扩大**：AMD 和 Blackwell 相关 PR 将逐步进入主线，社区会关注新卡的实测性能数据。
