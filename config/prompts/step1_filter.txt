## 任务
从以下仓库数据中，筛选出**最值得深入了解**的 Issue 和 PR。

## 筛选标准
1. **技术趋势相关**：新模型支持、性能优化、新功能、Kernel 优化、分布式推理、架构重构、API 演进
2. **高互动讨论**：评论数多且尚未关闭的讨论（说明是当前热点）
3. **重大变更**：代码改动量大、影响范围广的 PR
4. **忽略**：bug 修复、CI/文档、版本更新、依赖升级、测试相关、refactor（除非影响范围广）

## 输出格式
仅输出 JSON 格式的选择结果，不要其他内容。示例格式：
```json
{

    "issues": [{"number": 12345, "reason": "简要说明"}],
    "prs": [{"number": 12346, "reason": "简要说明"}]
  }
}
```

- **精简原则**：每个仓库最多选择 2-3 个真正重要的 item

- 宁缺毋滥：只选择真正能反映技术趋势或重大变更的项目
